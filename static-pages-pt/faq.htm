Geral

O que é StepMania 5.3?

É a próxima versão do StepMania em desenvolvimento. Nosso objetivo é modernizar o código do existente StepMania 5.0.x/5.1 para melhorar sua performance e providenciar um melhor suporte para máquinas e OS modernos. Nós também estamos adicionando novas funções, incluindo as que forem requisitadas pela nossa comunidade, e também atualizar suporte para modos de jogo fora 4-key, adicionando mais modos de jogo e mais.

O que é o Project Moondance?

É uma comunidade para a Equipe Rizu (Equipe de desenvolvimento do Stepmania 5.3), que não só inclui 5.3 mas também outros projetos relacionados.

Meus charts/temas antigos/..etc ainda vão funcionar?

StepMania 5.3 é feito baseado no existente código do StepMania 5.0.12/5.1, e mantér compatibilidade é uma das nossa prioridades (De fato, essa é uma das razões de da nossa versão ser 5.3 e não subir um número inteiro como 6.0). Qualquer conteúdo que funcionava com StepMania 5.0.12/5.1 (incluindo charts, temas e NoteSkins) ainda devem funcionar com StepMania 5.3. Se você notar algum problema, por favor reporte.

Um pequeno comentário para desenvolvedores

Please note that we are deprecating the use of semicolon-separated cmd() blocks (i.e.  OnCommand=cmd(linear,1.5;diffusealpha,1); ) as a means of placing commands on actors in Lua scripts, as this method is a non-standard Lua function that was created for StepMania, and 5.3 is not optimized with this coding style in mind (themes may experience performance penalties if they use this method). It is strongly recommended that function chaining (i.e. code such as OnCommand=function(self) self:linear(1.5):diffusealpha(1) end, ) be used instead.

Posso instalar junto com outras versões do StepMania?

Sim. Nosso instalador no Windows usa o modo portátil, para que a data de perfis seja armazenada somente na pasta de instalação da versão em uso.

Se você quer compartilhar uma lista de charts com várias versões do StepMania, você pode usar a preferência AdditionalSongFolders no Save/Preferences.ini para especificar um caminho.

Vai existir suporte online?

O existente "Stepmania Online" (SMO) está atualmente obsoleto, mas nós temos planos futuros para desenvolver um modo online totalmente novo.

Cade o open source?/Por que vocês não são open source?

Atualmente nós não estamos mantendo o repositório público, mas nós vamos disponibilizar para o público no futuro. (Com o nosso código usando a licença Apache License 2.0). Como StepMania 5.3 está em desenvolvimento de uma forma extremamente rápida, suportar open source iria causar problemas de um usúario compilando uma build baseada em trabalho incompleto, oque seria difícil para nós de dar suporte.

Se você tiver qualquer questão relacionada ao código do StepMania 5.3 em sí, você é livre para perguntar para membros da equipe de desenvolvimento. Nós vamos estar felizes em discutir.

The project began as a method to add online and profiles to SM5, but when we looked into the engine to see how this could be appropriated, it was discovered just how much in a sorry state the codebase was, and it looked mostly abandoned. I spoke to Shakesoda in hopes of a resurrection in the form of a team project, and it was requested rather than breaking 10 years of content and community to take the time to make a proper 5.x build that is stable and supports what the community as a whole wants from the game.

This backward compatibility support has been an absolute git to do, but it's something now we are down the line which we are proud of, as it has brought many players and newcomers to the community, and provided a very stable experience for folks during the lockdown. We will continue to maintain a near 100% compatibility where we can, and this is taken into consideration whenever we plan a new feature or change engine components.

We asked if it was ok to close the source while this process was done, and it was deemed wise for the health of the repo and the need of people who are just wanting to play. Its why we've been around the repo but not on it as taint can happen and we do not wish this. SSC [Spinal Shark Collective, the 5.0.x development team] did this same behaviour before us, so we did not see it as a problem. Some vocal members of the community disagreed, including some of the SM dev team. We have also been compared to past SM repo contributors who may have left a less than savoury memory in the minds of some, and it has also been hard to differentiate from history we have no involvement with.

O que é StepMania 5.2 e por que vocês pularam essa versão?

5.2 originalmente estava em desenvolvimento para ser a próxima versão do StepMania. Porém, essa versão tinha um número grande de mudanças que quebrava suporte com contéudo de versões anteriores, e a comunidade/outros devs não gostaram disso.

5.2 originalmente iria ser 5.1. Porém, em resposta da recepção, ela foi oficialmente atrasada em favor da nova 5.1 que ainda era compativel com 5.0.12. Isso foi feito imaginando que a nova versão (Renomeada para 5.2) eventualmente estaria completa. Porém, isso acabou não acontecendo.

Simply Love não me deixa usar a versão 5.3

Simply Love 5.3 funciona na versão 5.3, porém a versão 4.8 do Simply Love bloqueia o StepMania 5.3 intencionalmente. Na versão 4.8 do tema, você pode apertar Escape 2 vezes para conseguir usar, mas qualquer versão acima dessa é bloqueada.

Essa proteção pode ser removida abrindo o seguinte arquivo: StepMania OutFox 5.3/Apperance/Themes/Simply Love/Scripts/SL-SupportHelpers.lua, mude a seguinte linha

if not (t[2] and (t[2]==0 or t[2]==1)) then return false end

para

if not (t[2] and (t[2]==0 or t[2]==1 or t[2]==3)) then return false end

Ou você pode usar esse fork não oficial que foi criado pela organização TinyFoxes, que contem funcionalidades e compatibilidade com o StepMania OutFox 5.3

Eu tenho um processador rápido e/ou recente e meu jogo está tendo vários problemas, por que isso?

Algumas partes do jogo são tão bem otimizadas que agora elas estão indo rápidas demais e as outras partes do jogo tem que tentar alcançar. Nós vamos resolver esse problema com o passar do tempo, nós recomendamos abrir algum programa em segundo plano pode resolver o problema temporariamente.

macOS

Eu estou tendo problemas para rodar o jogo.

Se você receber um erro assim como "No noteskins found" ou outro problema similiar, você vai ter que mudar as configurações de segurança do macOS para poder rodar o jogo. O sistema não confia em aplicativos não licenciados e prevente acesso a arquivos externos a não ser que seja dado permissão manual pelo usuário.

Para resolver isso, abra o Terminal, e execute o seguinte comando: xattr -dr com.apple.quarantine /caminho/do/jogo

Alternativamente, você pode excluir o caminho, e mover o arquivo que contém o jogo para a janela do Terminal.

Em outras versões, macOS Catalina também gera uma mensagem dizendo que o jogo está pedindo permissão para detectar suas teclas.

O jogo está renderizando em uma janela pequena no canto da tela.

Isso é porque o renderizador não suporta telas que escalam. Para resolver isso, clique com botão direito no executavel e selecione "Pegar Informação" (Ou destaque o arquivo e aperte Comando (⌘)+I, e ative "Abrir em Resolução Baixa" em General.
